# iofund-agent

Personal AI assistant for getting more value out of an I/O Fund subscription. Long-term goal: a polished POC pitchable to the I/O Fund team as a member-facing product.

## Stack

- **App:** Next.js 16 App Router · React 19 · TypeScript · pnpm · Tailwind v4 (two-theme warm-dark/-light token system) · `next/font` (Fraunces / Hanken Grotesk / Spline Sans Mono) · `recharts` (portfolio donut)
- **Auth:** Neon Auth (powered by Better Auth) — Google OAuth + email/password
- **DB:** Neon Postgres via Vercel Marketplace · Drizzle ORM · `@neondatabase/serverless`
- **LLM:** Vercel AI Gateway — `"anthropic/claude-sonnet-4-6"` for chat, article distillation, and the weekly digest. AI SDK v6 (`streamText` + `stepCountIs(5)`). Chat is multimodal: a user can attach/paste a portfolio screenshot and the model reads it inline (tickers + share counts) to call `analyze_portfolio_gap`. (The standalone `generateObject` vision-extraction pipeline was removed 2026-06-21 — see the portfolio-gap convention below.)
- **Prices:** Yahoo Finance public chart endpoint (no API key, no auth). Called by the `analyze_portfolio_gap` chat tool + the `/portfolio` review UI to compute live weights.
- **Storage:** Hybrid — Postgres for structured rows (trades, article metadata + **full distilled body** (FTS-indexed and rendered live), encrypted IOF credentials, IOF current positions); git for prose (strategy/thesis docs, weekly digests) plus a hand-curated `positions-bootstrap.yaml` snapshot. (Article bodies moved Postgres-only on 2026-06-19 — they used to be committed as `data/articles/*.md`; that dir is gone.)
- **Search:** Postgres `tsvector` full-text search on `articles(title, body)` with GIN index; pgvector RAG deferred to Phase 1.
- **Cron:** GitHub Actions + Python `scripts/` for trade poll (#2), article ingest (#3), weekly digest (#4), portfolio ingest (daily — authoritative positions refresh from IOF's portfolio PDF). Resend live (`onboarding@resend.dev` → Hunter's email) — the only outbound email from the system.

The thesis pair (agent + human) is kept in sync automatically by the weekly digest's drift check; the strategy pair is static (no auto-updater — IOF's process rarely changes).

  - `io-fund-portfolio-pie.pdf` — categorized current holdings + weight%, snapshot 2026-05-19. Source artifact for `positions-bootstrap.yaml`.
  - `io-fund-portfolio-history.pdf` — per-tranche entry history for currently-held positions (cost basis + entry date per tranche).
  - `io-fund-portfolio.pdf` — older snapshot, reference only.
  - `positions-bootstrap.yaml` — hand-curated snapshot of IOF's held tickers with category + weight%. **Now a manual fallback**, superseded by `scripts/ingest_portfolio.py` (daily auto-refresh). Read by `scripts/bootstrap_positions.py` (Task #4.5). Note: the *pie-chart* PDF's weights are a raster (unextractable), which is why this YAML used to be the only weight source — but the separate *table* PDF (`…-Portfolio.pdf`, version-agnostic match) carries per-ticker allocation %, theme, and company as real text, which `ingest_portfolio.py` parses, so manual transcription is no longer the primary path.
  - `digests/*.md` — written weekly by `scripts/digest_week.py` (Task #4); each file is a structured 5-section summary of that week's trades + articles.
- `scripts/` — Python cron entrypoints + one-shot maintenance tools. `tests/` holds pytest unit tests for the pure ingest logic (state machine, PDF parser, payload parsing) against captured fixtures — no network/DB/LLM; run with `cd scripts && python -m pytest`.
  - `ingest_trades.py` — RSS-less trade poll via Firebase + `__NEXT_DATA__` parser; upserts `public.trades` keyed by `iof:<server-id>`. Piggybacks: each newly-inserted trade triggers `update_position_from_trade` against `public.positions` (BUY → held, SELL+close/stop hit → closed, SELL+trim/half → held, HEDGE/COVER-HEDGE skipped). Then `enrich_positions` backfills metadata on any held position still missing it (new tickers IOF bought after the last bootstrap snapshot): **company** from Yahoo Finance `meta.longName` (no key) and **category** from a single batched AI Gateway classification into the bootstrap theme taxonomy. Fills NULLs only (never clobbers authoritative bootstrap values), marks the row `source='trade_replay+enriched'`, and leaves `baseline_weight_pct` NULL — weight is not auto-derivable (lives only in IOF's rasterized pie chart) and stays null until the authoritative bootstrap refresh. Best-effort: enrichment failure never blocks trade ingest; needs `AI_GATEWAY_API_KEY` for the category step (company-only if absent).
  - `ingest_articles.py` — daily RSS poll → diff vs `articles` table → Firebase-auth fetch → Sonnet 4.6 distillation via AI Gateway → write row to Postgres (body included; no git file, `distilled_path` NULL). Idempotent. Filters non-analytical posts two ways: a title regex at discovery (webinar replays / invitations / "no webinar" notices) and an LLM `SKIP` catch-all for administrative content the title filter misses.
  - `backfill_article_voice.py` — one-shot voice cleanup for existing `articles.body` rows (Postgres). Rewrites each distilled body to follow `WRITING.md` (plain voice, no "IOF"/"the book"/analyst names/em-dash tells) **preserving every fact, number, and ticker**; validates each rewrite and dumps a gitignored backup before writing. Needs only `DATABASE_URL` + `AI_GATEWAY_API_KEY` (no re-fetch). `--dry-run`, `--slug` (repeatable), `--limit` flags. Run after changing the distillation prompt's voice.
  - `bootstrap_positions.py` — one-shot **manual fallback** (superseded by `ingest_portfolio.py`). Reads `data/positions-bootstrap.yaml`, cross-references `first_entry_date` from `trades`, UPSERTs `public.positions`. Idempotent. `--dry-run` flag.
  - `ingest_portfolio.py` — **authoritative daily positions refresh.** Firebase-auths, scrapes the current Portfolio *table* PDF URL off `/premium/portfolio` (Prismic-hosted; the content-hash URL changes each refresh so it's re-discovered each run), downloads + extracts text via `pypdf`, parses per-ticker company + allocation % + theme deterministically (ticker anchored to the known-ticker set; theme normalized to the canonical taxonomy incl. `AI Semis`), validates allocations sum to ~100% before writing, and UPSERTs `public.positions` with `source='portfolio_pdf:<run-date>'` (company, category, `baseline_weight_pct`, `status='held'`). No LLM/vision — pure text parse. `--dry-run` flag. This is what finally fills `baseline_weight_pct` automatically.
  - `digest_week.py` — weekly digest. Reads past-7d trades + articles; generates 5-section digest via Sonnet 4.6; commits to `data/digests/`; runs a second LLM call to detect drift in `io-fund-thesis.agent.md`, and if found a third call regenerates the human `io-fund-thesis.md` page from the same evidence in its own voice — a single PR proposes edits to both docs so they stay in sync. Emails summary via Resend. Owns the full pipeline (DB → LLM → git → PR → email) so the workflow stays thin. `DIGEST_DRY_RUN=1`, `DIGEST_FORCE_OVERWRITE=1`, `DIGEST_SKIP_PR=1`, `DIGEST_SKIP_EMAIL=1` for granular local testing.
- `.github/workflows/` — `poll-trades.yml` (cron */30 weekdays), `discover-articles.yml` (cron 14:00 UTC daily, commits to main), `ingest-portfolio.yml` (cron 14:30 UTC daily, authoritative positions refresh from the IOF portfolio PDF), `weekly-digest.yml` (cron `0 21 * * 5` Friday 5pm ET, opens PRs updating both thesis docs — agent + human page — if drift detected), `hello.yml` (secrets smoke test).

## Development

### Setup

```bash
cd chat
pnpm install
npx vercel link                              # one-time: links chat/ to the iofund-agent Vercel project
npx vercel env pull .env.local --environment=production
```

Required env vars in `chat/.env.local`: `DATABASE_URL`, `NEON_AUTH_BASE_URL`, `NEON_AUTH_COOKIE_SECRET`, `IOF_CREDS_ENCRYPTION_KEY`, `AI_GATEWAY_API_KEY`. Neon Marketplace auto-injects the `DATABASE_*` vars; the others were added manually to the Vercel project.

### Running

```bash
cd chat && pnpm dev
```

predev copies `data/*.md` → `chat/_data/` so chat tools can read them. Hits port 3000 or next free.

### DB schema changes

```bash
cd chat
pnpm db:generate                              # diff db/schema.ts → new migration SQL
pnpm db:migrate                               # apply to Neon
```

`tsvector` generated columns and GIN indexes are hand-extensions to the Drizzle-generated migration (Drizzle's schema snapshot doesn't track them, which is fine — Postgres manages the generated column).

### Verification

- `cd chat && pnpm build` — type-check + production build.
- `cd chat && pnpm eval` — eval suite against the chat agent. Asserts on tool-call traces, not free-form prose. Preeval runs copy-data.sh.
- `cd scripts && python -m pytest` — pure-logic unit tests for the Python ingest pipeline (position state machine, portfolio-PDF parser, payload parsing). No network/DB/LLM.
- For UI changes, hit dev server in a browser. Type-check passes but doesn't catch rendering issues.

## Conventions and guardrails

- **IOF subscription content is paid material.** Never reproduce article prose verbatim in code, docs, or chat output. The distilled docs in `data/` are transformative summaries; new article ingestion follows the same rule (system prompt enforces this for the chat agent).
- **All user-facing prose follows `WRITING.md`** (repo root): the voice target + AI-tells checklist for the strategy/thesis pages, digests, distilled summaries, chat replies, UI microcopy, and the LLM prompts that generate them. Key rule: the internal repo name "IOF" never appears in user-facing prose — use "I/O Fund" / "the fund" / "the firm". Tone is set in the prompt, not patched in the output.
- **Trades source of truth is Postgres (`public.trades`)**, not the seed CSV/JSON files in `data/`.
- **Positions source of truth is Postgres (`public.positions`)**, kept current by three layers: (1) the trade-poll piggyback in `ingest_trades.py` keeps `status` in lockstep with `trades`; (2) `enrich_positions` (same cron) fills provisional company (Yahoo) + category (LLM) on brand-new tickers within 30 min; (3) `ingest_portfolio.py` (daily) overwrites all three metadata fields — company, category, **`baseline_weight_pct`** — authoritatively from IOF's portfolio table PDF. **`source` column encodes provenance**: `portfolio_pdf:<date>` = authoritative (current primary path); `bootstrap_yaml:<date>` = authoritative manual fallback; `trade_replay` = born from a trade, not yet enriched; `trade_replay+enriched` = provisional (Yahoo company + LLM-guessed category, overwritten by the next portfolio-PDF run). Precedence holds because enrichment only fills NULLs and only upgrades `source` from `trade_replay`, never from `portfolio_pdf`.
- **Personal portfolio holdings come only from a screenshot the user attaches in chat.** The model reads tickers + share counts from the image inline and calls `analyze_portfolio_gap({ holdings })`; per-share prices + values are NEVER used — weights are computed live from Yahoo Finance server-side, which sidesteps the "is the screenshot showing price or value?" ambiguity. The attached image is persisted in `chat_messages` so it survives a reload in the user's own thread; nothing in the request path is stored as a separate holdings row. (The dedicated upload UX — onboarding step, `/portfolio/holdings`, `lib/portfolio/extract.ts`, `/api/portfolio/*` — was removed 2026-06-21 in favor of this in-chat flow. The `public.user_holdings` table was dropped in migration `0007`.)
- **Two-layer auth:** Neon Auth identifies the app user; `public.iof_credentials` stores their AES-encrypted IOF email+password keyed to `neon_auth.user.id`. **Never put `IO_FUND_USERNAME`/`IO_FUND_PASSWORD` in Vercel env vars.** They live only in GH Actions secrets (for cron jobs that run as Hunter) and in encrypted DB rows (for end-user IOF subscriptions).
- **`IOF_CREDS_ENCRYPTION_KEY` is unrotatable** — losing or changing it bricks every stored IOF credential. Rotation requires a re-encrypt migration.
- **Citations are render-layer, not prompt-driven.** The Sources block in `chat-thread.tsx` is built deterministically from the `read_article` tool-call trace. The model writes free-form prose; the renderer guarantees attribution. Don't add citation rules to the system prompt.
- **Don't auto-commit changes to the strategy/thesis docs** (`io-fund-{strategy,thesis}.md` or their `.agent.md` pairs). The weekly digest opens an auto-PR to a branch (thesis docs only); Hunter approves manually. The daily article cron writes only to Postgres now — it makes no git commits (as of 2026-06-19; previously it committed `data/articles/*.md` to main).
- **Don't deploy publicly without IOF's blessing.** Private Vercel URL only until the pitch lands.

## End-goal product

Three hero features (Phase 0 target):

1. **Natural-language chat** over IOF subscription content (live as of 2026-05-19; FTS retrieval + render-layer sources as of 2026-05-20).
2. **Weekly auto-digest** of new IOF activity (live as of 2026-05-20; opens auto-PRs updating both thesis docs — agent + human page — when drift detected).
3. **Personal portfolio gap analysis** (live diff vs IOF current book) — the user attaches/pastes a portfolio image in chat, the model extracts holdings inline and calls `analyze_portfolio_gap({ holdings })` (live 2026-06-20). The earlier dedicated screenshot-upload UX (saved to `user_holdings`, live 2026-05-23) was removed 2026-06-21 in favor of the single in-chat flow.

Architecture is **multi-tenant-clean from day one** — every per-user table keyed by `user_id`. Phase 3 multi-tenant rollout adds Postgres RLS policies, not an auth/data-model rewrite.
